SSH SSM Spring SpringMvc
Spring是java EE領域中來實踐企業級應用的開發框架，能夠搭建複雜的網站後端服務
Spring 兩個特性(因此降低開發的複雜度)

  1. 控制反轉(IOC)

    • 不是什麼新的技術，只是一個設計思想，IOC可以讓我們擺脫複雜的物件管理工作，像是在傳統開發時候常會用new來建立物件，每次在new都會創造物件之間的依賴關係，programmer必須去維護這些工作，所以可以將我們設計好的物件交給他這第三方的IOC容器來管控，總結就是IOC容器可以讓我們有效率的建立物件以及組裝物件之間的依賴關係=>XML配bean

  2. 面相切面程式(AOP)

    • 實現代碼解偶的一個重要利器，使我們可以在不改變程式碼(功能流程)的狀況下，加入新的功能，這是一種況展性的表現，他的基本原理就是”動態代理技術”

Spring有個缺點就是隨著項目越來越複雜或龐大時候，就會需要引入很多配置文件(XML)，難以理解且容易出錯
Spring 開發一個web項目有幾個步驟

  1. 配置maven依賴
  2. 配置web.xml 加載Spring mvc
  3. 配置DB連接或配置Spring Application
  4. 配置加載文件的讀取 開啟註解
  5. 配置日誌文件 接著配置一堆…
  6. 最後部屬Tomcat調適

=======================================
Spring boot不是為了取代Spring，而是為了讓開發者更方便簡化的使用Spring，首先就先將Spring裡面複雜的XML配Bean給砍掉。
Spring boot核心思想就是約定大於配置(Convention over Configuration, CoC)，表示developer只要定義應用中會有不符合Spring boot約定的部分來做調整就ok，其他大部分約定俗成的東西Spring boot都幫我們配置好了

Spring boot搞一個web項目，只要把maven依賴都加進來，基本上就是可以直接啟動，XML複雜配置Spring boot全部幫我們搞定，
所以Spring booot有兩項優點

  1. Spring boot有很多開箱即用的套件(jar)，像是各式各樣的Starter (EX: spring-boot-starter-actuator)
  2. 配置直接簡化
  3. 部屬，內簽Tomcat容器，啟動和部屬變得相當容易
